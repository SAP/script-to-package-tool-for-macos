Script2Pkg

Umgang mit doppelten Paketnamen

Wenn Sie ein neues Paket erstellen und ein Paket mit demselben Namen bereits am ausgewählten Speicherort existiert, wird Script2Pkg das neue Paket umbenennen, um den Namenskonflikt zu beseitigen. Sie können dieses Standardverhalten so verändern, dass das existierende Paket stattdessen überschrieben wird.

 

Um existierende Pakete mit demselben Namen bei der Paketerstellung zu überschreiben, gehen Sie bitte wie folgt vor:

 

1. Wählen Sie im Menü  Script2Pkg  den Menüpunkt  Einstellungen …

 

 

2. Wählen Sie die Option  Speicherorte .

3. Wählen Sie aus dem Menü  Falls Paket bereits existiert:  die Option  Existierendes Paket überschreiben  aus.

 

Standardeinstellung wiederherstellen

Um die Standardeinstellung wiederherzustellen, wählen Sie bitte im Menü  Falls Paket bereits existiert:  die Option  Neues Paket umbenennen  aus.
