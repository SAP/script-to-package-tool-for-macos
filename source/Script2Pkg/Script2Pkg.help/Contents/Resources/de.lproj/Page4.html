Script2Pkg

Es gibt Begriffe, die in Script2Pkg verwendet werden und die möglicherweise einer Erklärung bedürfen. Im Folgenden finden Sie die Gebräuchlichsten:

 

Installationspaket

Ein Installationspaket ist eine Datei mit der Dateierweiterung  .pkg , die unter macOS verwendet wird. Sie enthält normalerweise ein Archiv mit zu installierenden Dateien und Informationen darüber, wo sie zu installieren sind. Sie kann auch zusätzliche Ressourcen wie Anleitungen oder Lizenzdokumente sowie Skripte enthalten, die vor oder nach der Installation ausgeführt werden. Man unterscheidet zwischen zwei Arten von Installationspaketen:

  • Komponentenpakete: Dies sind die Standard-Installationspakete, die ein Archiv mit den zu installierenden Dateien und die Informationen darüber enthalten, wo die Dateien installiert werden sollen.

  • Verteilungspakete: Diese Pakete können ein oder mehrere Komponentenpakete enthalten und außerdem noch Ressourcen zur Anpassung und Steuerung der im macOS Installationsprogramm angezeigten Benutzeroberfläche. 

  •  

Nutzlastfreies Paket

Nutzlastfreie Pakete sind Installationspakete für macOS, die nur dazu dienen, Skripte auszuführen. Sie installieren keine Dateien, die als Nutzlast des Installationspakets bezeichnet werden. Da in diesen Installationspaketen keine Nutzlast enthalten ist, werden die von dieser Anwendung erstellten Installationspakete als  nutzlastfrei  bezeichnet.

 

Beglaubigung

Die Beglaubigung ist ein zusätzlicher Schutz davor, dass signierte Installationspakete von einer böswilligen dritten Partei verändert werden. Der Apple Notary Service ist ein automatisiertes System, das ein Installationspaket auf bösartigen Inhalt und auf Probleme bei der Codesignierung überprüft. Die Beglaubigung eines signierten Installationspakets stellt also sicher, dass Apple den signierten Code durch einen automatisierten Scan-Prozess geleitet und überprüft hat, dass der signierte Code frei von bösartigem Inhalt ist. Die in macOS integrierten Sicherheitswerkzeuge können den Beglaubigungsstatus eines signierten und beglaubigten Installationspakets überprüfen.

 

Paketquittung

Paketquittungen werden von macOS verwendet, um festzustellen, welche Pakete vom Installationsprogramm installiert wurden. Normalerweise hinterlässt die Installation jedes Installationspakets eine Paketquittung. Nutzlastfreie Pakete bilden hier allerdings eine Ausnahme, da sie keinerlei Dateien installieren, nicht einmal eine Paketquittung. Paketquittungen enthalten folgende Informationen: 

 

  • Paketkennung: Eine Zeichenfolge, die zur eindeutigen Identifizierung eines Installationspakets verwendet wird.

  • Paketversion: Eine Zeichenkette, die zum Speichern von Versionsinformationen für das Installationspaket verwendet wird.

 

Hinweis:  Es gibt eine Option, die es ermöglicht, dass von dieser Anwendung erstellte nutzlastfreie Pakete bei der Installation eine Quittung hinterlassen.
