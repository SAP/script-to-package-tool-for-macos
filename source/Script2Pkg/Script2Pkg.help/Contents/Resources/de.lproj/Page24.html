Script2Pkg

Ordnerbasierte Pakete

Möglicherweise wollen Sie einmal ein Installationspaket erstellen, in dem Sie dem Skript, das Sie paketieren wollen, zusätzliche Ressourcen zur Verfügung stellen müssen. Bei diesen Ressourcen kann es sich um ein zusätzliches Skript, eine Programm oder sogar ein anderes Installationspaket handeln. Script2Pkg bietet eine entsprechende Option, für die folgende Voraussetzungen gelten:

 

Sie müssen einen Ordner erstellen. In diesem Ordner muss sich  mindestens ein Skript  mit folgendem Namen befinden:

 

  • preinstall

  • postinstall

 

Diese Skripte müssen  ausführbare Dateien  sein. Der Grund dafür ist, dass zwei Kategorien von Skripten für die Verwendung mit Installationspaketen definiert wurden:

 

  • preinstall : Das Preinstall-Skript wird ausgeführt, bevor Dateien installiert werden. Wenn das Skript keinen Exit-Status von Null zurückgibt, bricht Installer die Installation ab.

  • postinstall : Das Postinstall-Skript wird ausgeführt, nachdem die Dateien installiert wurden. Wenn das Skript keinen Exit-Status von Null zurückgibt, meldet das macOS Installationsprogramm die Installation als fehlgeschlagen.

 

Die Namen der Skripte müssen genau wie beschrieben lauten, da das macOS Installationsprogramm nur ausführbare Dateien mit diesen Namen als Skripte verwenden kann, die Aktionen vor und nach der Installation ausführen können. Wenn die Dateinamen anders lauten, werden sie vom macOS Installationsprogramm nicht erkannt oder verwendet.

 

Sobald Sie alle Elemente zur Verfügung haben, erstellen Sie das Paket wie folgt:

 

1. Starten Sie die Anwendung.

2. Wenn Sie das zu erstellende Paket signieren möchten, stellen Sie bitte sicher, dass die Option  Pakete signieren  aktiviert ist und aktivieren Sie zusätzlich die Option  Pakete beglaubigen , wenn Sie ein beglaubigtes Paket erstellen wollen.

3. Klicken Sie auf die Taste  Skripte auswählen .

4. Wählen Sie, in dem nun erscheinenden Fenster, den Ordner aus, der die Komponenten für Ihr Paket enthält und klicken Sie anschließend auf die Taste  Erstellen . In diesem Beispiel ist neben dem  Postinstall-Skript  auch noch das Programm  jq  enthalten. Dies ermöglicht es dem  Postinstall-Skript , das Programm  jq  direkt aus dem Paket heraus auszuführen, ohne dass es vorher installiert werden muss. 

5. Das Installationspaket wird nun erstellt und am selben Speicherort abgelegt, an dem sich auch der ausgewählte Ordner befindet. Das erzeugte Paket trägt den Namen des ausgewählten Ordners.
